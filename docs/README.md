# Docs Map

This folder has one problem: too many documents with different authority levels.

Use this order:

1. `ROADMAP_ENGINE_TO_ENTITY.md`
   Forward-plan authority. What wave comes next and why.
2. `PROJECT_STATE.md`
   Audited runtime truth. What is actually landed and validated.
3. `WORKLIST.md`
   Current execution lane and immediate next actions.
4. `MISSING_FEATURES_REGISTRY.md`
   Active gaps only.
5. `SESSION_INDEX.md`
   Session ledger. Use it to find the matching session log.

## Folders

- `sessions/`
  Session-by-session execution records. Start with `SESSION_INDEX.md`, then open only the specific session log you need.
- `specs/`
  Decision-complete implementation contracts for bounded waves and subsystems.
- `research/`
  Source-backed exploration and horizon work. Informative, not authoritative by default.
- `agent_briefs/`
  Lane-specific coordination rules for parallel work.

## Root Docs

- `PHASE_EXECUTION_COMPANION.md`
  Execution-grade companion for the roadmap when waves are actively being worked.
- `REPO_STRUCTURE.md`
  Repo layout reference.
- `REFERENCE_AUDIT.md`
  Audit notes on code/docs references.
- `REFERENCE_INSIGHTS.md`
  Supporting insights from earlier review passes.

## Legacy / Secondary

- `MASTER_PLAN_EXPANDED.md`
- `MASTER_PLAN_EXPANDED_OLD.md`
- `TODO.md`

These are historical or secondary planning artifacts. Do not treat them as execution authority over the roadmap/state/worklist/registry set.
