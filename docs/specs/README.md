# Specs Guide

These files are implementation contracts for bounded subsystems or waves.

## How To Use This Folder

- read the spec that matches the active wave before changing that subsystem
- treat a spec as design authority for that lane unless the roadmap/state docs explicitly supersede it
- do not open every spec by default; pick the one tied to your task
