# Sessions Guide

This folder is the execution history, not the first place to orient yourself.

Use this order:

1. Open `../SESSION_INDEX.md`.
2. Find the session ID relevant to the wave or file you care about.
3. Open only that matching session log.

## What Lives Here

- one markdown file per session
- `_template.md` for new session logs

## How To Read It

- newer files are not automatically more authoritative
- use the session goal and result to decide whether the log matters
- prefer sessions that match the active wave or the file you are touching

## Current Pattern

- `2026-03-06-*`
  Wave 2 through Wave 7 governance, runtime, packaged proof, and parity work
- `2026-03-07-*`
  Wave 3 onward implementation, UI structure planning, diagnostics cleanup, and later-wave expansion

If you are trying to understand current repo truth, read `../PROJECT_STATE.md` and `../WORKLIST.md` first, then use this folder for evidence and exact change context.
