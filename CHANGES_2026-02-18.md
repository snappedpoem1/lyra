# Lyra Oracle - System Changes Summary

## Changes Made (2026-02-18)

### 1. Switched to Music-Specific CLAP Model
Changed from `laion/clap-htsat-unfused` to `laion/larger_clap_music`:
- Better music-specific embeddings
- Auto-fallback to smaller model if larger fails to load
- Configurable via `LYRA_CLAP_MODEL` env var

Files updated:
- `oracle/embedders/clap_embedder.py` - New model with fallback logic
- `oracle/indexer.py` - Updated MODEL_NAME
- `oracle/scorer.py` - Updated MODEL_NAME  
- `oracle/search.py` - Updated MODEL_NAME

**Note:** First run after this change will download the new model (~600MB). Existing embeddings remain compatible.

### 2. Simplified Acquisition Waterfall (3 Tiers)
Cleaned up to just 3 working tiers:

| Tier | Source | Quality | Requires |
|------|--------|---------|----------|
| T1 | Real-Debrid | FLAC | Prowlarr + RD API key |
| T2 | Slskd | FLAC | Slskd running |
| T3 | SpotDL | 320k MP3 | spotdl installed |

All tiers **gracefully degrade** when services aren't available.

### 3. Fixed Real-Debrid Integration
Created clean, direct RD API module (`acquirers/realdebrid.py`):
- No qBittorrent/RDT-Client dependency
- Direct API flow: Search → Check cache → Add torrent → Poll → Download
- Instant availability check for cached torrents
- Direct HTTPS download of unrestricted files

### 4. Auto-Score on Ingest
Indexer now automatically scores tracks after embedding:
```bash
oracle pipeline --library "A:\music\Active Music"  # Scans, indexes, AND scores
oracle index --no-score  # Skip auto-scoring if needed
```

### 5. New CLI Commands
- `oracle status` - Quick system overview (row counts + tier availability)
- `oracle acquire waterfall --artist X --title Y` - Run full acquisition cascade

## Configuration (.env)

```ini
# === T1: Real-Debrid (FLAC torrents) ===
PROWLARR_URL=http://localhost:9696
PROWLARR_API_KEY=your_prowlarr_key
REAL_DEBRID_KEY=your_rd_api_key

# === T2: Slskd (P2P FLAC) ===
# Optional - gracefully skips if not running
LYRA_PROTOCOL_NODE_URL=http://localhost:5030
LYRA_PROTOCOL_NODE_USER=slskd
LYRA_PROTOCOL_NODE_PASS=slskd

# === T3: SpotDL (YouTube fallback) ===
# Just needs: pip install spotdl
# No config needed

# === CLAP Model ===
# Optional - defaults to laion/larger_clap_music
LYRA_CLAP_MODEL=laion/larger_clap_music
HF_HOME=C:/MusicOracle/hf_cache

# === Write Safety ===
LYRA_WRITE_MODE=apply_allowed  # or: readonly, plan_only
```

## Usage Examples

```powershell
# Check system status
oracle status

# Run full pipeline (scan + index + score)
$env:LYRA_WRITE_MODE="apply_allowed"
oracle pipeline --library "A:\music\Active Music"

# Acquire a track (tries T1 → T2 → T3)
oracle acquire waterfall --artist "Daft Punk" --title "Digital Love"

# Check tier availability
python -m oracle.acquirers.waterfall

# Test Real-Debrid connection
python -m oracle.acquirers.realdebrid
```
